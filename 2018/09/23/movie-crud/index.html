2. 添加电影

1. 在forms.py中建立MovieForm,并根据相应字段填写描述
2. 在views.py中创建MovieForm的实例，并传入模板进行渲染
3. 在views.py中接收表单传过来的数据，并进行相应的逻辑修改

2.1 MovieForm的创建

示例： 

class MovieForm(FlaskForm):
"""电影管理表单"""
title = StringField(
    label="片名",
    validators=[
        DataRequired("请输入片名！")
    ],
    description="片名",
    render_kw={
        "class": "form-control",
        "id": "input_title",
        "placeholder": "请输入片名！",
    }
)

其中，label和render_kw可以传入模板代替<label>和<input>标签，而validators的作用是判断上传数据是否为空。若为空，可通过以下代码在网页中渲染出错误：

{% for err in form.title.errors %}
    

    {{ err }}
    

    {% endfor %}

其中，wtforms的不同字段可以替换不同的标签：

1.PasswordField 替换密码输入框
2.SubmitField 替换提交按钮
3.FileField 替换文件上传框
4.TextAreaField 替换文本框
5.SelectField 替换单选框，需要添加参数：
    coerce=int, # 设置限定数据类型
    choices=[(v.id, v.name) for v in tags], # 设置选项

2.2 MovieForm的实例创建及模板渲染

实例创建示例：

form = MovieForm()    
render_template("admin/movie_add.html", form=form)

模板渲染示例：

<label for="input_logo">{{ form.logo.label }}</label>
{{ form.logo }}

其中，仅替换文本内容，替换整个input标签。

2.3 数据的接收与其逻辑判断

示例：

if form.validate_on_submit():
    # 取出数据
    data = form.data
    # 得到上传文件的文件名    
    file_url = form.url.data.filename
    file_logo = form.logo.data.filename
    # 是否存在该文件夹，不存在则创建
    if not os.path.exists(app.config['UP_DIR']):
        os.makedirs(app.config['UP_DIR'])
        os.chmod(app.config['UP_DIR'], 'rw')
    # 改变文件名并保存到文件夹
    url = change_filename(file_url)
    logo = change_filename(file_logo)
    form.url.data.save(app.config['UP_DIR'] + url)
    form.logo.data.save(app.config['UP_DIR'] + logo)
    # 修改数据
    movie = Movie(
        title=data["title"],
        url=url,
        info=data['info'],
        logo=logo,
        star=int(data['star']),
        play_num=0,
        comment_num=0,
        tag_id=int(data['tag_id']),
        area=data['area'],
        release_time=data['release_time'],
        length=data['length'],
    )
    # 保存数据
    db.session.add(movie)
    db.session.commit()
    flash("添加电影成功！", "ok")
    return redirect(url_for("admin.movie_add"))

其中，分为数据的得到，数据的修改和数据的保存。若需要保存上传文件的地址，需要设置app.config['UP_DIR']。flash的闪现消息需要在模板中渲染，如下：

{% for msg in get_flashed_messages(category_filter=["ok"]) %}
    

    	{{ msg }}
   	

    {% endfor %}

3. 电影列表

1. 在views.py中设置页面信息
2. 在movie_list.html中渲染模板

3.1 页面信息的创建

示例：

# 分页信息
page_data = Movie.query.join(Movietag).filter(
    Movietag.id == Movie.tag_id
).order_by(
    Movie.addtime
).paginate(page=page, per_page=10)

其中，query是查询，join是连接相关表，filter是连结条件，order_by是排序规则，paginate是flask_sqlalchemy的分页函数。

3.2 模板的渲染

示例（提取出movie的信息）：

{% for v in page_data.items %} 
		{{ v.id }}
	{% endfor %}

示例（分页模板的搭建）：

{% macro page(data, url) %}
	{% if data %}
	
    
	    
  • 首页
    • 
	    {% if data.has_prev %}
	    
  • 上一页
    • 
	    {% else %}
	    
  • 上一页
    • 
	    {% endif %}
	
	    {% for v in data.iter_pages() %}
	        {% if v == data.page %}

    • 
	        {% else %}
	        
  • {{ v }}
    • 
	        {% endif %}
	    {% endfor %}
	
	    {% if data.has_next %}
	    
  • 下一页
    • 
	    {% else %}
	    
  • 下一页
    • 
	    {% endif %}
	
	    
  • 尾页
    • 
	

	{% endif %}
	{% endmacro %}

分页模板使用：

1.先导入
    {% import "ui/admin_page.html" as pg%}
2.相应的div中调用
    {{ pg.page(page_data, 'admin.movie_list')}}

其中，page_data是分页信息，admin.movie_list是路由规则。

4.电影删除

1. 在模板中设置删除按钮
2. 在views.py中增加删除逻辑

4.1 设置删除按钮

根据id来删除电影：

<a href="{{url_for('admin.movie_del', id=v.id)}}" class="label label-danger">删除</a>

4.2 增加删除逻辑

示例：

def movie_del(id=None):
    movie = Movie.query.get_or_404(id)
    db.session.delete(movie)
    db.session.commit()
    flash("删除电影成功", "ok")
    return redirect(url_for("admin.movie_list", page=1))

其中，先根据id查询到相关电影，并在数据库中删除，闪现消息可以在movie_list.html中显示，再重定向到movie_list界面。

5. 电影编辑

1. 通过id找到相关电影，并传入新创建的模板：movie_edit.html
2. 当修改数据并提交后，对相应数据进行修改后保存

5.1 movie_edit.html的创建

与添加页面相似，但需要渲染movie的初始值。方法如下：

1. 根据form模板建立，设置value值：
    {{ form.area(value=movie.area) }}
2. 原生方法，在视图函数中判断请求方法并直接设置初始值：
    if request.method == "GET":
        form.info.data = movie.info
        form.tag_id.data = movie.tag_id
        form.star.data = movie.star

5.2 修改逻辑

示例：

if form.validate_on_submit():
    data = form.data
    movie_count = Movie.query.filter_by(title=data["title"]).count()
    if movie_count == 1 and movie.title != data["title"]:
        flash("片名已经存在！", "error")
        return redirect(url_for("admin.movie_edit", id=id))

    if not os.path.exists(app.config['UP_DIR']):
        os.makedirs(app.config['UP_DIR'])
        os.chmod(app.config['UP_DIR'], 'rw')

    if form.url.data.filename != "":
        file_url = form.url.data.filename
        movie.url = change_filename(file_url)
        form.url.data.save(app.config['UP_DIR'] + movie.url)

    if form.logo.data.filename != "":
        file_logo = form.logo.data.filename
        movie.logo = change_filename(file_logo)
        form.logo.data.save(app.config['UP_DIR'] + movie.logo)

    # 修改
    movie.title = data["title"]
    movie.star = data["star"]
    movie.tag_id = data["tag_id"]
    movie.info = data["info"]
    movie.area = data["area"]
    movie.length = data["length"]
    movie.release_time = data["release_time"]
    # 保存
    db.session.add(movie)
    db.session.commit()
    flash("修改电影成功", "ok")
    return redirect(url_for("admin.movie_edit", id=id))

其中，上传文件名的修改需要在初始化movie时设置：

form.url.validators = []
form.logo.validators = []
