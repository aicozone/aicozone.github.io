远程Linux主机配置

发表于 2018-05-23

前期准备

  • 云服务器 (我选择的是阿里云的学生套餐)
  • github for desktop
  • bitvise ssh client

##安装过程##

###生成ssh_keys###

安装github for desktop 后, 打开git shell并输入以下命令:

ssh-keygen -C mykey

此处的mykey是一个标注, 用于自己的记忆.

后续会提示ssh_key存储位置, 默认即可; 还会提示要不要为这个key存储密码, 没有必要, 两个回车即可. 最后, 在那个存储位置里, 你可以看到两个key, 一个是私钥, 用于自己保管; 另一个是公钥, 用于各处分享.

###在主机和bitvise ssh client处配置ssh-keys###

  1. 在Linux主机处配置公钥

    使用用户名和密码登入主机

    1. 用 root 用户登录到服务器, 创建 .ssh 目录
    cd /root
    mkdir .ssh
2. 编辑 authorized_keys 文件, 把刚才生成的 id_rsa.pub 文件里面的内容粘贴进去并保存退出 (注意, 这里可以粘贴多个 key, 一行一个)
    nano .ssh/authorized_keys
  2. 在bitvise ssh client处配置私钥 (具体见此处)

###各类软件初始化###

  1. 服务器初始化配置, 复制这些命令粘贴到服务器终端中执行

    # 安装 配置 打开 ufw 防火墙
apt-get install ufw      # 安装
ufw allow 22             # 允许端口
ufw allow 80
ufw allow 443
ufw default deny incoming
ufw default allow outgoing
ufw status verbose  
ufw enable

  2. 安装必备软件

    apt-get install git python3 python3-pip python3-setuptools supervisor mongodb redis-server zsh

wget https://github.com/robbyrussell/oh-my-zsh/raw/master/tools/install.sh -O - | sh

pip3 install gunicorn

    注意: 安装过程中若遇到失败, 可以尝试升级系统

    apt-get update

  3. 服务器中文编码问题

    # 编辑下面的文件, 不要拼错
nano /etc/environment
# 加入下面的内容, 保存退出
LC_CTYPE="en_US.UTF-8"
LC_ALL="en_US.UTF-8"

最后, 如果你注册了域名的话, 可以将该主机与域名绑定, 这样, 后续的网站建设工作便可以顺利进行了.
